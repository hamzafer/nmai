## Error Analysis

1. **"Et bilag kan ikke registreres uten posteringer"** = voucher has no postings
2. **"Posteringene på rad 0 er systemgenererte"** = row 0 postings are system-generated and cannot be created externally

This means the voucher `postings` format is still wrong. The LLM is likely sending:
- `debitAmount`/`creditAmount` (wrong) instead of `amount` (positive=debit, negative=credit)
- Or the postings array structure doesn't match what Tripletex expects

## What Would Fix This

1. Fix voucher posting format — must use `amount` field (positive=debit, negative=credit)
2. Each posting needs: `{"account": {"id": N}, "amount": N, "description": "..."}`
3. Total of all `amount` fields must sum to zero
4. The "system-generated" error suggests accounts 1720 or 1029 may have auto-posting rules
