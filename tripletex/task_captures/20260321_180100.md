## Root Cause Analysis

Three issues compounding:

1. **Missing credit posting knowledge:** The system prompt docs for `/supplierInvoice` don't explain that voucher postings must be balanced (debit + credit = 0). The LLM knows it needs credit but doesn't know the correct structure.

2. **Accounts payable account (2400) lookup failed:** GET /ledger/account?number=2400 returned 422 instead of the account. The auto-fallback range search didn't trigger because the response wasn't 200 with empty values — it was a 422. Need to try range search: `?numberFrom=2400&numberTo=2410`.

3. **VAT handling in postings is wrong:** For supplier invoices with input VAT:
   - Debit expense account for NET amount (67,050) with vatType for 25% input VAT
   - Tripletex should auto-generate the VAT posting if vatType is set correctly
   - OR: manually add debit to VAT account (2710) for VAT amount (16,762.50)
   - Credit AP account (2400) for GROSS amount (-83,812.50) with no VAT type
